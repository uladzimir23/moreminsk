# ADR-002 — Next.js Static Export

## Контекст

Проект — лендинг/контентный сайт с ~28 страницами. Сервер-сайд рендеринг (SSR) или инкрементальная регенерация (ISR) не требуются для MVP:
- Контент меняется раз в месяц-квартал
- Нет персонализации
- Нет личного кабинета
- Формы — отдельный микросервис или внешний сервис

Главные требования — **SEO**, **скорость**, **дешёвый хостинг**.

## Решение

Используем **Next.js 16 App Router + `output: "export"`** — полностью статический билд.

## Последствия

### Позитивные

- Максимально быстрая отдача (HTML уже готов, никаких вычислений на запрос).
- Хостинг на любом CDN/nginx/S3 — бесплатные/дешёвые варианты (Vercel/CF Pages/Netlify бесплатный tier перекрывает нужды MVP).
- Идеально для SEO — нет JS-гидрации блокирующей первый рендер.
- Нулевые runtime-зависимости сервера.

### Негативные

- **Нет API routes** — формы нужно слать наружу (Telegram Bot / Resend / Formspree).
- **Нет middleware** — редиректы по `Accept-Language` недоступны, только клиентские.
- **Нет ISR / revalidate** — каждое обновление контента = redeploy.
- **`next/image` optimizer отключён** (`images: { unoptimized: true }`) — оптимизация через sharp на билд-стадии.
- **Нет preview-режима** next-intl в той форме, что с middleware.

### Что не критично (но стоит знать)

- Нет server-only данных (нет секретов в бандле, т.к. всё публично).
- Build time растёт с количеством страниц, но для ~100 страниц это < 1 мин.

## Альтернативы (отвергнутые)

1. **Next.js SSR на Vercel/Node** — избыточно, добавляет точки отказа.
2. **Astro** — отличная альтернатива для контентного сайта, но у исполнителя в экосистеме Next.js больше опыта и готовый шаблон (clariva).
3. **Gatsby** — устарел, поддержка падает.
4. **Плоский HTML + Webflow/Tilda** — быстро, но ограничения на кастом и SEO-тонкости.

## Триггеры пересмотра

Вернуться к этому решению, если:
- Появится онлайн-бронирование с календарём занятости (нужен real-time)
- Появится личный кабинет клиента
- Требуется A/B-тестирование на серверной стороне
- Нужна мультитенантность

В этом случае — переход на SSR на Vercel или гибрид (static + edge functions).
