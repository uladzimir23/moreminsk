# Meta-стратегия

> Формулы для `<title>` и `<meta description>`. Черновые значения — согласовать с заказчиком перед публикацией.

## Правила

### Title
- **Длина:** 50–60 символов (до обрезки в выдаче)
- **Формула:** `{Ключ} — {УТП/уточнение} | Море Minsk`
- **Ключ — в начале**, бренд — в конце
- Цифры работают лучше, чем без («от 150 BYN»)
- Избегать: «»*()— в мета (ломает парсеры некоторых агрегаторов)

### Description
- **Длина:** 140–160 символов
- **Структура:** `[Что] [для кого] [УТП] [CTA]`
- Содержит ключ + 1–2 LSI-термина + цену + CTA
- НЕ повторяет title дословно
- CTA в конце: «Забронируйте онлайн», «Звоните +375...», «Свободные даты в Telegram»

### Keywords
- ❌ **Не используем.** Тег `<meta keywords>` игнорируется поисковиками с 2009. См. [[../10 - Brief & Requirements/Бриф клиента]] — заказчик явно просил удалить.

## Open Graph / Twitter Cards

Каждая страница:
- `og:title` = title без `| Море Minsk`
- `og:description` = description
- `og:image` — 1200×630, уникальное изображение для страницы (не logo)
- `og:type` = `website` (главная) / `article` (блог) / `product` (яхта)
- `og:locale` = `ru_RU` (и `en_US` если EN)
- `twitter:card` = `summary_large_image`

## Реализация в Next.js

В каждой странице через `export async function generateMetadata()`:

```typescript
import type { Metadata } from 'next';
import { generatePageMetadata } from '@/lib/seo';

export async function generateMetadata(): Promise<Metadata> {
  return generatePageMetadata({
    title: 'Свадьба на яхте в Минске — от 1500 BYN',
    description: 'Свадебная церемония и банкет...',
    path: '/services/svadba',
    ogImage: '/og/svadba.jpg',
  });
}
```

См. `lib/seo.ts` — утилиты в [[../40 - Architecture/Architecture Overview]].
