# Медиа-стратегия

## Цель

Продаёт визуал. Наша задача — дать потенциальному клиенту **убедиться** в атмосфере/качестве до звонка.

## Источники контента (в порядке приоритета)

### 1. Instagram @moreminsk.by (существующий)
- Выкачать все фото/видео за 2 последних года
- Категоризовать: по яхте, по мероприятию
- Выбрать топ-30 фото (с разрешением ≥ 1920px)
- Получить согласие клиентов на публикацию (если лица)

### 2. Собственная съёмка (рекомендованная)
Одна производственная смена закрывает большую часть нужд:
- **День** — 4–6 часов на воде
- **Команда:** фотограф + видеограф + капитан
- **Модели:** 2–3 пары друзей/знакомых
- **Сценарии:** загрузка на борт, выход, парус, напитки, смех, закат, возвращение
- **Результат:** 80+ фото, 4 видео (по яхте) + 1 общее

### 3. Стоки — НЕ используем
Заказчик явно подчёркивал — «продаёт только визуал». Стоки убивают доверие.

### 4. UGC (User Generated Content)
Пост-MVP: промо «Отметь нас в сторис — 10% к следующему заказу» → лента живого контента.

## Что нужно снять для MVP

### Фото (per yacht × 4 яхты)
- 1× главное с воды (под парусом / на ходу)
- 1× интерьер сверху
- 1× кокпит с людьми
- 1× деталь: штурвал / парус / якорь
- 1× швартовка у пирса
- 3–5× мероприятие на борту (день рождения, закат)

**Итого:** ~30 фото на 4 яхты = ~120 фото.

### Видео
- 4× вертикальное видео-обход яхты (30–60 сек) — для TikTok/Reels + сайта
- 1× горизонтальное общее (1–2 мин) — для Hero на главной

### OG-картинки
- 1× главная (1200×630)
- 4× по яхтам (1200×630)
- 15× по услугам (1200×630) — можно на базе одного шаблона в Figma
- 1× /galereya
- 1× /ceny

**Итого:** ~21 OG-картинка.

## Технические требования

### Фото
- Raw → JPEG export
- Минимум 2400px по большей стороне
- Сохранить EXIF (дата съёмки — полезно для Schema.org)
- Поставщик отдаёт обработанные JPEG + cover-crops

### Видео
- MP4 H.264, 1080p или 4K
- Вертикаль: 1080×1920 (9:16)
- Горизонталь: 1920×1080
- Без водяных знаков

## Оптимизация на билде

Скрипт `scripts/optimize-images.ts` через `sharp`:

```
Источник: /raw-media/fleet/eva-01.jpg (3000×2000, 2.5MB)

Генерирует:
  /public/fleet/eva-01.webp          (1920×1280, ~150KB)
  /public/fleet/eva-01@1024.webp     (1024×683, ~60KB)
  /public/fleet/eva-01@640.webp      (640×427, ~25KB)
  /public/fleet/eva-01.avif          (современный формат)
```

В JSX:
```tsx
<picture>
  <source type="image/avif" srcSet="/fleet/eva-01.avif" />
  <source type="image/webp" srcSet="
    /fleet/eva-01@640.webp 640w,
    /fleet/eva-01@1024.webp 1024w,
    /fleet/eva-01.webp 1920w
  " />
  <img src="/fleet/eva-01.webp" alt="Яхта EVA на Минском море" loading="lazy" decoding="async" />
</picture>
```

Для LCP-фото (Hero) — `loading="eager"` + `fetchpriority="high"`.

## Alt-тексты — правила

- Каждое фото имеет `alt`
- Alt содержит ключ + контекст:
  - ✅ `Свадьба на яхте ALFA на Минском море, июль 2025`
  - ❌ `alfa-01.jpg`
  - ❌ `Красивая яхта на воде`
- Не пишем «фото» или «изображение» в alt — это и так ясно

## Видео — placement

| Где | Что | Длина |
|---|---|---|
| Главная — Hero | Горизонтальное общее | 30–60 сек, autoplay muted loop |
| Карточка яхты | Вертикальный обход | 30 сек, clickable preview |
| /galereya | Все видео сеткой | по 30–60 сек |

**Вес видео:** не больше 3MB на файл (через HandBrake/ffmpeg → H.264 CRF 23).

## Водяные знаки и копирайт

- НЕ ставим водяные знаки на свои фото (ухудшает визуал)
- В `<meta name="copyright">` — бренд
- Если клиент не разрешил лицо — кропим или blur
