# Анимации — спецификация

> Решение 2026-04-17: берём **рекомендованный набор** (баланс premium ↔ performance). Все анимации уважают `prefers-reduced-motion` — альтернатива через opacity only, без transform/scale.

## Базовые принципы

1. **Только Framer Motion** (~30kb) — не GSAP, не Lottie (ADR-006).
2. **Все durations и easings — через токены** из `design-system/tokens/_motion.scss`. Никаких хардкод-значений.
3. **Один эффект = один смысл.** Не мешаем scale + rotate + fade одновременно. Если нужен «вау» — используем multi-layer shadow или glow, не анимацию.
4. **Mobile-first budget.** На mobile каждая анимация стоит FPS — парадокс «всё летает» выше `lg+`, на mobile — базовые fade-up.
5. **prefers-reduced-motion:** `transform: scale/translate → opacity only`, durations → `--duration-fast`.

## Токены (уже есть в DS)

```scss
/* durations */
--duration-instant: 50ms;
--duration-fast:    150ms;  /* tap feedback */
--duration-base:    200ms;  /* hover/focus */
--duration-medium:  300ms;  /* modal/page */
--duration-slow:    500ms;  /* sheet open */
--duration-slower:  800ms;  /* hero stagger total */

/* easings */
--ease-spring:  cubic-bezier(0.22, 1, 0.36, 1);    /* default */
--ease-sheet:   cubic-bezier(0.32, 0.72, 0, 1);    /* Apple sheet */
--ease-out:     cubic-bezier(0.16, 1, 0.3, 1);     /* fast start, slow end */
--ease-bounce:  cubic-bezier(0.34, 1.56, 0.64, 1); /* attention only */
--ease-linear:  linear;                             /* skeleton/pulse */
```

## 1. Анимации для ТЕКСТА

### 1.1 Staggered fade-up (Hero H1 + lead)

**Где:** заголовок H1 Hero + lead-абзац + CTA-кнопки.
**Как:** родитель `motion.div` с `variants={container}`, дети с `variants={item}`, stagger 0.08–0.12s.

```tsx
const container = {
  hidden: { opacity: 0 },
  show: {
    opacity: 1,
    transition: { staggerChildren: 0.1, delayChildren: 0.1 },
  },
};
const item = {
  hidden: { opacity: 0, y: 24 },
  show: {
    opacity: 1,
    y: 0,
    transition: { duration: 0.6, ease: [0.22, 1, 0.36, 1] },
  },
};
```

**На mobile:** stagger 0.06s (быстрее — экран меньше, внимание короче).
**Reduced motion:** `y: 24 → 0` заменяется на fade only.

### 1.2 Underline swipe для `<Accent>`

**Где:** accent-слова в H1 и section-titles (компонент `<Accent>` из `shared/ui/accent`).
**Как:** псевдо `::after` с `transform: scaleX(0 → 1)`, `transform-origin: left`, `--duration-slow`, `--ease-out`. Триггер — `useInView({ once: true })`.

```scss
.accent {
  position: relative;
  &::after {
    content: '';
    position: absolute;
    inset-inline: 0;
    inset-block-end: 0.05em;
    block-size: 0.12em;
    background: color-mix(in oklch, var(--color-accent) 35%, transparent);
    transform: scaleX(0);
    transform-origin: left;
    transition: transform var(--duration-slow) var(--ease-out);
  }
  &.is-visible::after { transform: scaleX(1); }
}
```

### 1.3 Number counter

**Где:** квик-стата в Hero («127 мероприятий», «8 лет на воде», «4.9 ★»).
**Как:** `framer-motion` `useMotionValue` + `useTransform` + `animate()` от 0 до target за 1.2s, `--ease-out`. Триггер — `useInView`.
**Форматирование:** `.toFixed(0)` для целых, `.toFixed(1)` для рейтинга.

```tsx
const count = useMotionValue(0);
const rounded = useTransform(count, v => Math.round(v));
useEffect(() => {
  if (inView) animate(count, target, { duration: 1.2, ease: [0.16, 1, 0.3, 1] });
}, [inView]);
```

**Reduced motion:** показываем финальное значение без анимации.

## 2. Анимации для БЛОКОВ / СЕКЦИЙ

### 2.1 Section fade-up on scroll (базовый)

**Где:** все widgets-секции (`hero`, `fleet-grid`, `services-grid`, `price-table`, `gallery`, `reviews`, `faq`, `cta-form`, `why-us`, `contacts`).
**Как:** родитель `motion.section` с `useInView({ once: true, threshold: 0.15 })`, `y: 32 → 0`, `opacity: 0 → 1`, `--duration-medium`, `--ease-spring`.
**Исключение:** Hero НЕ анимируется как секция (он первый экран — не ждём скролл). Hero имеет свою внутреннюю staggered-анимацию (§1.1).

### 2.2 Parallax фото (desktop only)

**Где:** Hero image, gallery hero-shots (первое большое фото в lightbox-галерее яхты).
**Как:** `useScroll()` + `useTransform(scrollY, [0, 500], [0, -60])` → `transform: translateY(var)`. Только при `breakpoint >= lg`.
**Mobile:** отключено (performance).
**Reduced motion:** отключено.

### 2.3 Sticky-reveal (опционально, только Price Table)

**Где:** блок `widgets/price-table` на `/ceny` при горизонтальном скролле таблицы.
**Как:** первая колонка таблицы имеет `position: sticky; inset-inline-start: 0`, при scroll правой части — видна всегда.
**Это чистый CSS** (не framer-motion). Смотри §3.1 в этом документе не для обсуждения — уже решено.

### 2.4 Floating card idle-bobbing

**Где:** floating-card поверх hero-фото («3 свободные даты», «Отвечаем за 15 мин»).
**Как:** `animate={{ y: [0, -6, 0] }}`, `transition={{ duration: 4, repeat: Infinity, ease: 'easeInOut' }}`.
**Reduced motion:** отключено, статично.

## 3. Анимации для КНОПОК

### 3.1 Tap-scale 0.97 (базовый)

**Где:** все `<Button>` (primary, secondary, ghost, outline, icon). Плюс карточки `<Card>` с `clickable` prop.
**Как:** `transform: scale(0.97)` on `:active`, `transition: transform var(--duration-fast) var(--ease-spring)`. Чистый CSS через mixin `mx.tap-scale`.
**Reduced motion:** `transform: scale(0.99)` (минимальный feedback сохраняем).

### 3.2 Shimmer sweep по primary CTA

**Где:** primary-кнопки «Забронировать» в Hero и на карточках яхт.
**Как:** `::before` с `background: linear-gradient(110deg, transparent 40%, rgba(255,255,255,0.3) 50%, transparent 60%)`, `transform: translateX(-100% → 100%)`, `--duration-slower` (800ms), `--ease-out`. Интервал между шиммерами — **8 секунд** (чтобы не раздражать).
**Условие:** только если кнопка **в viewport** (через `useInView`), иначе анимация идёт впустую.
**Reduced motion:** отключено.

### 3.3 Icon morph

**Где:**
- `ThemeToggle` — иконки `Monitor → Sun → Moon → Monitor` с `AnimatePresence` (fade + rotate 180°).
- Close buttons (X в sheet/modal) — scale `0.8 → 1` на появлении.
- Accordion chevron — `rotate: 0 → 180` при открытии (через `data-state` Radix).

```tsx
<AnimatePresence mode="wait">
  <motion.div
    key={theme}
    initial={{ rotate: -90, opacity: 0 }}
    animate={{ rotate: 0, opacity: 1 }}
    exit={{ rotate: 90, opacity: 0 }}
    transition={{ duration: 0.2 }}
  >
    <Icon />
  </motion.div>
</AnimatePresence>
```

### 3.4 Pulse-dot на badge

**Где:** индикатор статуса на `<Badge>` («Свободно сегодня», «В наличии», «3 даты на май»).
**Как:** вложенный `<span>` с `animation: pulse 2s infinite ease-in-out`.

```scss
@keyframes pulse {
  0%, 100% { transform: scale(1); opacity: 1; }
  50%      { transform: scale(1.4); opacity: 0.5; }
}
```

Плюс второй `<span>` поверх — статичная точка. Получается эффект «волны» от точки.
**Reduced motion:** статичная точка без волны.

## 4. Анимации для ТРАНЗИТОВ (sheet/modal/page)

### 4.1 AppPanel (ADR-007)

**Mobile — bottom-sheet:**
- Open: `y: 100% → 0`, `--duration-slow` (500ms), `--ease-sheet`
- Close: `y: 0 → 100%`, `--duration-medium` (300ms), `--ease-sheet`
- Drag: `y` следует пальцу, dismiss при `y > 120px`

**Desktop — side-drawer:**
- Open: `x: 100% → 0`, `--duration-slow`, `--ease-sheet`
- Close: `x: 0 → 100%`, `--duration-medium`, `--ease-sheet`

**Backdrop:** fade `opacity: 0 → 1`, `--duration-medium`, параллельно.

### 4.2 Wizard step-transitions (Booking Module)

**Где:** переходы между шагами в `features/booking`.
**Как:** `AnimatePresence mode="wait"` + `x: 16 → 0` при next, `x: -16 → 0` при back. `--duration-base`, `--ease-out`.
**Содержимое:** панель остаётся открытой, меняется только содержимое слота.

### 4.3 Page transition

**Как:** при переходе между страницами в Next.js — **не делаем** кастомных page transitions (это мешает SEO crawling и LCP). Просто стандартный `router.push`.

## 5. Loading states

### 5.1 Skeleton (контент ещё грузится)

**Где:** не нужен в static export (весь контент — в HTML). Используется только для dynamic-секций (если появятся, например iframe карты до click-to-load).
**Как:** `background: linear-gradient(90deg, var(--color-surface-alt) 0%, var(--color-surface) 50%, var(--color-surface-alt) 100%)` + `animation: skeleton-pulse 1.4s ease-in-out infinite`.

### 5.2 Submit spinner (Booking wizard Step 5)

**Где:** кнопка «Отправить заявку» после клика, пока `submitBooking()` не ответил.
**Как:** inline-spinner Lucide `<Loader2 />` с `animation: spin 1s linear infinite`.
**Не закрываем кнопку** — оставляем текст «Отправляем...» + spinner слева.

## 6. Анимации, от которых отказались

| Эффект | Почему нет |
|---|---|
| Typewriter для H1 | SEO-риск (crawler видит пустой text), раздражает при reload |
| Split-text character reveal | Expensive на mobile, нужна отдельная библиотека |
| Ripple (Material) | Не iOS-стиль, не сочетается с capsule buttons |
| Cursor magnet | Spot-cool-but-annoying, breaks focus-trap accessibility |
| Custom page transitions (fade/slide between routes) | Мешает LCP, crawling, back-button UX |
| Scroll-triggered animations на каждом элементе | «Убивает» FPS, отвлекает от контента |
| Lottie illustrations | Весит 100+ KB, не iOS-стиль. Пока отказ — возможно для empty-state в post-MVP |

## 7. Implementation checklist

При реализации нового виджета/компонента:

- [ ] Все durations через `var(--duration-*)`, не хардкод ms
- [ ] Все easings через `var(--ease-*)`, не хардкод cubic-bezier
- [ ] Есть `@media (prefers-reduced-motion: reduce)` override
- [ ] На mobile эффект не «тяжелее» чем fade-up + scale
- [ ] `useInView` для scroll-триггеров использует `once: true` (не повторяется)
- [ ] Если анимация infinite — проверяем что отключается вне viewport (для CPU)
- [ ] Tap-scale применён ко всем clickable элементам через `mx.tap-scale`

---

## 8. Performance budget

| Метрика | Цель |
|---|---|
| JS framer-motion bundle | ≤ 40 KB gzip (только `motion/react`, не `lazy-features`) |
| Total animation CPU on mobile mid-range | < 5% при scroll |
| Hero staggered fade-up total | ≤ 800ms (от first frame до последнего) |
| LCP (Hero image) | < 2.5s — image без анимации, только content вокруг |
| CLS | 0.0 — все анимации через transform/opacity, не width/height |
